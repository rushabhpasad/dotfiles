# Rules
- Adhere strictly to internal rule files (security-checks.md, performance-guidelines.md, code-review-standards.md).
- Check generated code for compliance with local CI/CD, linting, and style configurations.
- Highlight any proposed changes that may violate local policies.
- Provide compliant alternatives wherever non-compliance is detected.
- Never override local policies unless explicitly instructed.
