# Code Review Rules
- Always check for readability, maintainability, and testability.
- Point out unclear variable names or magic numbers.
- Ensure consistent indentation, spacing, and comments.
- Identify any logic duplication or unused code.
- Highlight missing test coverage for new or modified logic.
- Prioritize clarity and maintainable design over micro-optimizations.
- Provide fixes in code blocks, with short explanations before them.
- Label severity levels as: ✅ Info, ⚠️ Moderate, ❌ Critical.
